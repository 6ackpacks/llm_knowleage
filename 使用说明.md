# 使用说明

## 目录结构

```
konwleadge/
├── $raw/        ← 把收集的原始资料放这里（Web Clipper 保存到这里）
├── $wiki/       ← LLM 自动生成的结构化知识库（不要手动编辑）
│   ├── index.md ← 自动维护的全库索引
│   └── RULES.md ← LLM 编译规范
├── $output/     ← LLM 生成的报告、图表、幻灯片
└── 使用说明.md
```

## 工作流

### Step 1：收集资料
用浏览器的 **Obsidian Web Clipper** 扩展，把感兴趣的网页保存到 `$raw/` 目录。

也可以直接把 PDF、文章复制进 `$raw/`。

### Step 2：让 Claude 编译 Wiki
打开终端，进入这个 Vault 目录，运行 Claude Code：

```bash
cd "C:\llm知识库\konwleadge"
claude
```

然后对 Claude 说：

> "请读取 $raw/ 下的所有新文件，把它们整理编译到 $wiki/ 中，更新 $wiki/index.md 索引。"

### Step 3：问答
当 $wiki/ 积累了足够多的文章后，直接问 Claude：

> "根据 $wiki/ 里的内容，帮我分析一下……"

Claude 会检索知识库，给出基于你自己收集内容的回答。

### Step 4：输出归档
让 Claude 把分析报告输出到 `$output/`，同时可以要求它把有价值的输出归档回 `$wiki/`，让知识库越来越丰富。

---

## 常用指令（直接对 Claude 说）

| 指令 | 作用 |
|------|------|
| "编译 $raw/ 下的新文件到 $wiki/" | 把新收集的资料整理进知识库 |
| "更新 $wiki/index.md 索引" | 重建全库索引 |
| "对 $wiki/ 做健康检查" | 找数据不一致、补充缺失内容 |
| "根据 $wiki/ 回答：[问题]" | 基于知识库问答 |
| "把这次分析归档到 $wiki/[主题].md" | 保存问答结果 |
